# 进程调度模拟系统实现方案

## 文件结构

```
caozuoxitong/
├── pcb.py          # PCB 数据类
├── algorithms.py   # 两种调度算法
└── main.py         # tkinter GUI 主程序
```

## 核心数据结构 — `pcb.py`

`PCB` 类字段：

- `name`、`priority`（当前/初始）、`arrival_time`、`burst_time`
- `used_cpu`（已用 CPU 时间）、`remaining`（剩余运行时间）
- `state`：`R`(Ready) / `E`(Execute) / `W`(Wait) / `F`(Finish)
- `start_time`、`finish_time`（记录周转时间用）
- `queue_level`（MLFQ 专用，标记所在队列编号 0/1/2）

## 调度算法 — `algorithms.py`

### Algorithm 1: 基于优先级的时间片轮转（Priority RR）

- 单就绪队列，按 `priority` 升序排列（数值小 = 优先级高）
- 每次 `step()` 执行一个时间片：
  - 到达时间 ≤ 当前时钟的进程加入就绪队列
  - 选队首进程执行（优先级最高者），状态设为 `E`
  - 执行完一个时间片后，`priority += decay`（decay 默认 2）
  - 若进程完成 → 状态 `F`；否则重新入队，按新优先级重排
  - 若有更高优先级进程到达，抢占当前进程

### Algorithm 2: 多级反馈队列（MLFQ）

- 3 个就绪队列（`Q0/Q1/Q2`），时间片大小为 `ts × 1`、`ts × 2`、`ts × 4`
- 新到达进程进入 `Q0`
- 执行逻辑：优先调度编号小的非空队列
- 未在当前时间片完成 → 降级到下一队列
- 同一队列内按 FIFO 顺序
- 高优先级队列非空时抢占低优先级队列的执行进程

两个调度器均暴露：

- `step() → event_log`：推进一个时间片，返回本步日志
- `is_done() → bool`：所有进程是否完成或死锁
- `get_stats() → list[dict]`：各进程开始/结束/周转时间

## GUI 布局 — `main.py`

使用 `tkinter.ttk.Notebook` 两个标签页（Priority RR / MLFQ），每页结构相同：

```
┌─────────────────────────────────────────────────────┐
│  [输入面板]  进程数/时间片/衰减值 + 进程参数表格     │
│  [控制按钮] 开始 | 单步 | 自动运行 | 重置           │
├──────────────────────┬──────────────────────────────┤
│  进程状态表 (Treeview)│  队列可视化 (Canvas)         │
│  名称/优先级/到达/   │  Priority RR: 就绪队列排列   │
│  已用/剩余/状态      │  MLFQ: Q0/Q1/Q2 三行         │
├──────────────────────┴──────────────────────────────┤
│  甘特图 (Canvas) — 时间轴 + 每进程彩色块             │
├─────────────────────────────────────────────────────┤
│  统计结果表 + 平均周转时间 + 两算法性能对比文字说明  │
│  调度日志滚动文本框                                  │
└─────────────────────────────────────────────────────┘
```

- **自动运行**：使用 `widget.after(speed_ms, step_callback)` 循环步进，支持速度调节滑块
- **甘特图**：每步在 Canvas 上追加色块（每进程分配固定颜色）
- **队列可视化**：每步用 Canvas 绘制方块列表，当前执行进程高亮

## 关键实现要点

- 输入验证：优先级 1~100 整数，到达时间 ≥ 0 整数，运行时间 ≥ 1 整数
- 死锁检测：就绪/执行队列均空但存在 `R/W` 状态进程时判定死锁
- 性能对比：两个算法用相同输入数据，在统计页并排显示平均周转时间
- 界面健壮：运行中禁用输入控件，异常输入弹出 `messagebox` 提示
