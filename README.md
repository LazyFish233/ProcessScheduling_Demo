# 进程调度模拟系统

基于 Python + tkinter 的进程控制与进程调度算法模拟实现，支持可视化动态演示调度过程。

## 功能概述

- **基于优先级的时间片轮转调度算法（Priority Round-Robin）**：单就绪队列按优先级排序，每执行一个时间片后优先级衰减，支持抢占。
- **多级反馈队列调度算法（MLFQ）**：3 级就绪队列（Q0/Q1/Q2），时间片逐级倍增，未完成进程自动降级，高优先级队列可抢占。
- **性能对比分析**：两种算法在相同输入下的平均周转时间对比与结论。

## 环境要求

- Python 3.10+
- tkinter（Python 标准库，无需额外安装）

## 使用说明

### 界面标签页

| 标签页 | 说明 |
|--------|------|
| 基于优先级的时间片轮转 | 设置进程参数和时间片大小，运行 Priority RR 算法 |
| 多级反馈队列 | 设置进程参数和基础时间片，运行 MLFQ 算法 |
| 性能对比 | 两种算法运行完毕后，刷新查看平均周转时间对比 |

### 操作步骤

1. 在进程参数表中添加/编辑进程（进程名、优先级 1~100、到达时间、运行时间）
2. 设置基础时间片大小和优先级衰减值（仅 Priority RR）
3. 点击 **开始** 初始化调度器
4. 点击 **单步** 逐步推进，或点击 **自动** 连续运行（可调节速度）
5. 观察进程状态表、队列可视化、甘特图和调度日志的动态变化
6. 调度完成后查看统计结果表和平均周转时间
7. 切换到 **性能对比** 标签页，点击刷新查看两种算法的对比结论

### 进程状态说明

| 状态 | 含义 |
|------|------|
| R（Ready） | 就绪，等待 CPU 分配 |
| E（Execute） | 正在执行 |
| W（Wait） | 等待（阻塞） |
| F（Finish） | 已完成 |

## 算法说明

### 基于优先级的时间片轮转

- 优先级数值越小，优先级越高
- 每执行完一个时间片，当前进程优先级值增加（衰减），防止高优先级进程长期占据 CPU
- 若新到达进程优先级高于当前执行进程，发生抢占

### 多级反馈队列（3 级）

- Q0 时间片 = base × 1，Q1 = base × 2，Q2 = base × 4
- 新到达进程进入 Q0
- 在当前队列时间片内未完成的进程降级到下一队列
- 始终优先调度编号最小的非空队列，高优先级队列有进程时抢占低优先级队列
